# SSR API Notes

Tiptap editors are client-side interactive components. In SSR frameworks like Next.js, render editor screens from client components. The package defaults immediate rendering to `false` for editor setup, JSON rendering, and preview rendering to reduce hydration mismatch risk.

## Client Component

```tsx
"use client";

import { RichTextEditor, useTiptapEditor } from "tiptap-react-ui";

export function Editor() {
  const { editor, isEditorLoading } = useTiptapEditor();

  if (isEditorLoading) return null;

  return <RichTextEditor editor={editor} />;
}
```

## Key Props

| API | Prop | Recommended SSR Value | Why |
| --- | --- | --- | --- |
| `useTiptapEditor` | `immediatelyRender` | `false` by default | Avoids rendering the editor before the client is ready. |
| `useTiptapEditor` | `isEditorLoading` | Check before rendering editor UI | Lets you show a loading state while the editor instance is created. |
| `RichTextEditor` | `immediatelyRenderPreview` | `false` by default | Avoids preview hydration mismatch. |
| `RenderJSON` | `immediatelyRender` | `false` by default | Defers JSON rendering until client-side mount. |

## Preview Example

```tsx
<RichTextEditor
  editor={editor}
  enablePreview
/>
```

## RenderJSON Example

```tsx
<RenderJSON content={content} />
```

For editor-generated saved content, prefer JSON and render it with `RenderJSON`. Use HTML only when you specifically need serialized markup.

## RenderHTML Note

`RenderHTML` renders an HTML string and does not create an editable editor instance. If the content comes from users or external systems, pass a sanitizer before rendering.
