# useTiptapEditor Hook

The `useTiptapEditor` hook provides a pre-configured Tiptap editor instance with all extensions already set up, eliminating the need for manual configuration. This hook simplifies editor initialization and returns a ready-to-use editor instance along with utility functions for managing content.

## Overview

Instead of manually configuring extensions and editor options, the `useTiptapEditor` hook handles all the setup internally. You can immediately start using the editor with sensible defaults while still having access to advanced customization options.

## Import

```tsx
import { useTiptapEditor } from "tiptap-react-ui";
```

## Signature

```ts
function useTiptapEditor(options?: TiptapEditorProps): TiptapEditorReturnTypes;
```

## Options

`useTiptapEditor` accepts the selected `UseEditorOptions` supported by the package, plus library-specific options.

| Option              | Type                                             | Default           | Description                                                   |
| ------------------- | ------------------------------------------------ | ----------------- | ------------------------------------------------------------- |
| `content`           | `Content`                                        | -                 | Initial editor content as HTML or Tiptap JSON.                |
| `autofocus`         | `boolean \| "start" \| "end" \| "all" \| number` | -                 | Controls initial focus behavior.                              |
| `editable`          | `boolean`                                        | -                 | Controls whether the editor can be edited.                    |
| `immediatelyRender` | `boolean`                                        | Tiptap default    | Set to `false` in SSR/client-rendered frameworks when needed. |
| `editorProps`       | `EditorProps`                                    | Internal defaults | ProseMirror editor props. The library sets base CSS classes.  |
| `onUpdate`          | `Function`                                       | -                 | Called when editor content changes.                           |
| `onCreate`          | `Function`                                       | -                 | Called when the editor instance is created.                   |
| `onFocus`           | `Function`                                       | -                 | Called when the editor receives focus.                        |
| `onBlur`            | `Function`                                       | -                 | Called when the editor loses focus.                           |
| `isPreview`         | `boolean`                                        | `false`           | Removes edit-mode padding and border for preview surfaces.    |
| `className`         | `string`                                         | -                 | Adds classes to the ProseMirror editor element.               |
| `handleImageUpload` | `object`                                         | fallback uploader | Configures uploads for the image upload node.                 |

## Return Value

```ts
type TiptapEditorReturnTypes = {
  editor: Editor;
  getEditorContent: () => {
    json: JSONContent | null;
    html: string | null;
  };
};
```

| Property           | Type                   | Description                                                                                                     |
| ------------------ | ---------------------- | --------------------------------------------------------------------------------------------------------------- |
| `editor`           | `Editor`               | The Tiptap editor instance. It can be unavailable during initial mount, so guard before calling editor methods. |
| `getEditorContent` | `() => { json, html }` | Returns current content as Tiptap JSON and serialized HTML. Returns `null` values when the editor is not ready. |

## Usage Examples

### Basic Setup

```tsx
import { RichTextEditor, useTiptapEditor } from "tiptap-react-ui";

export function Editor() {
  const { editor, getEditorContent } = useTiptapEditor({
    content: "<p>Hello world</p>",
  });

  return <RichTextEditor editor={editor} />;
}
```

### With Image Upload

```typescript
const { editor, getEditorContent } = useTiptapEditor({
  content: "<p>Welcome to the editor</p>",
  handleImageUpload: {
    onUpload: async (file, onProgress) => {
      // Simulate upload
      for (let i = 0; i <= 100; i += 10) {
        await new Promise((r) => setTimeout(r, 100));
        onProgress?.({ progress: i });
      }
      return "/images/uploaded.jpg";
    },
    maxLimit: 10 * 1024 * 1024,
    limit: 20,
  },
});
```

### With Lifecycle Hooks

```typescript
const { editor, getEditorContent } = useTiptapEditor({
  onUpdate: ({ editor }) => {
    const { json, html } = getEditorContent();
    console.log("Content updated:", { json, html });
  },
  onFocus: () => {
    console.log("Editor focused");
  },
  onBlur: () => {
    console.log("Editor blurred");
  },
});
```

### Preview Mode

```typescript
const { editor } = useTiptapEditor({
  content: savedContent,
  isPreview: true,
  className: 'prose prose-sm',
});

return (
  <div className="p-4 bg-white rounded-lg">
    {/* Content displays in read-only preview format */}
  </div>
);
```

### Custom Styling

```typescript
const { editor } = useTiptapEditor({
  content: "<p>Styled editor</p>",
  className: "h-96 p-4 border-2 border-blue-500 rounded-lg",
});
```

### Getting Editor Content

```typescript
const { editor, getEditorContent } = useTiptapEditor({
  content: "<h1>My Document</h1><p>Some content</p>",
});

function handleSave() {
  const { json, html } = getEditorContent();

  // Send to your API
  fetch("/api/documents", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      content: json,
      html: html,
    }),
  });
}
```

## Important Notes

- **Null Handling**: The `editor` can be `null` during the initial mount. Always check before using it.
- **SSR**: For Next.js SSR environments, wrap the component with `'use client'` directive.
- **Performance**: The `getEditorContent()` function can be called frequently, but consider debouncing expensive operations like API calls.
- **Abort Signals**: Use the provided `abortSignal` in image uploads to properly handle cancellations.
- **Content Formats**: Both JSON and HTML formats are always available. Use JSON for data storage and HTML for rendering.
