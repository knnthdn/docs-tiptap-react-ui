# useTiptapEditor Hook

The `useTiptapEditor` hook creates a preconfigured Tiptap editor instance with the extensions used by `RichTextEditor`, `NotionEditor`, `RenderJSON`, and the package preview surfaces.

It returns the editor instance, a loading flag, and helpers for reading the current content as JSON and HTML.

## Import

```tsx
import { useTiptapEditor } from "tiptap-react-ui";
```

## Signature

```ts
function useTiptapEditor(options?: TiptapEditorProps): TiptapEditorReturnTypes;
```

## Options

`useTiptapEditor` accepts selected Tiptap `UseEditorOptions`, plus library-specific options.

| Option              | Type                                             | Default                | Description                                                                                     |
| ------------------- | ------------------------------------------------ | ---------------------- | ----------------------------------------------------------------------------------------------- |
| `content`           | `Content`                                        | -                      | Initial editor content as HTML or Tiptap JSON.                                                  |
| `autofocus`         | `boolean \| "start" \| "end" \| "all" \| number` | -                      | Controls initial focus behavior.                                                                |
| `editable`          | `boolean`                                        | -                      | Controls whether the editor can be edited.                                                      |
| `immediatelyRender` | `boolean`                                        | `false`                | Controls immediate editor rendering. Defaults to `false` to reduce SSR hydration mismatch risk. |
| `editorProps`       | `EditorProps`                                    | internal defaults      | ProseMirror editor props. The library sets base attributes and merges your options.             |
| `placeholder`       | `string`                                         | `"Write something..."` | Placeholder text shown when the editor is empty.                                                |
| `className`         | `string`                                         | -                      | Adds classes to the editable ProseMirror editor element.                                        |
| `handleImageUpload` | `object`                                         | fallback uploader      | Configures uploads for the image upload node.                                                   |
| `tableOfContents`   | `TableOfContentsOptions`                         | -                      | Registers and configures the Tiptap table of contents extension.                                |

## Return Value

```ts
type TiptapEditorReturnTypes = {
  editor: Editor;
  isEditorLoading: boolean;
  getEditorContent: () => {
    json: JSONContent | null;
    html: string | null;
  };
};
```

| Property           | Type                   | Description                                                                                                     |
| ------------------ | ---------------------- | --------------------------------------------------------------------------------------------------------------- |
| `editor`           | `Editor`               | The Tiptap editor instance. It can be unavailable during initial mount.                                         |
| `isEditorLoading`  | `boolean`              | `true` while `useEditor` has not created an editor instance yet.                                                |
| `getEditorContent` | `() => { json, html }` | Returns current content as Tiptap JSON and serialized HTML. Returns `null` values when the editor is not ready. |

## Basic Setup

```tsx
"use client";

import { RichTextEditor, useTiptapEditor } from "tiptap-react-ui";

export function Editor() {
  const { editor, isEditorLoading } = useTiptapEditor({
    content: "<p>Hello world</p>",
  });

  if (isEditorLoading) return null;

  return <RichTextEditor editor={editor} />;
}
```

## Default Styling

The hook applies minimal editor defaults:

```txt
tiptap text-left text-base focus:outline-0
```

It does not impose minimum height, maximum height, overflow, padding, borders, or background styles, it naturally adopt its wrapper and grow with content. Add those through `className` when you need to style the actual editable ProseMirror document.

```tsx
const { editor } = useTiptapEditor({
  className: "min-h-[420px] max-h-[70vh] overflow-y-auto p-6",
});
```

## With Image Upload

```tsx
const { editor } = useTiptapEditor({
  handleImageUpload: {
    maxLimit: 10,
    limit: 5,
    onUpload: async (file, onProgress, abortSignal) => {
      const formData = new FormData();
      formData.append("file", file);

      const response = await fetch("/api/upload", {
        method: "POST",
        body: formData,
        signal: abortSignal,
      });

      onProgress?.({ progress: 100 });
      const data = await response.json();
      return data.url;
    },
  },
});
```

## Getting Editor Content

```tsx
const { editor, getEditorContent } = useTiptapEditor({
  content: "<h1>My Document</h1><p>Some content</p>",
});

function handleSave() {
  const { json, html } = getEditorContent();

  if (!json || !html) return;

  saveDocument({ json, html });
}
```

For persisted editor content, prefer storing and rendering JSON with `RenderJSON`. Store HTML alongside it only when you need display-ready markup, indexing, email output, or external integrations.

## Important Notes

- Use `isEditorLoading` when you want to render a loading state while the editor initializes.
- For Next.js and other SSR frameworks, keep editor components in client components.
- `immediatelyRender` defaults to `false` for SSR safety.
- `getEditorContent()` can be called frequently, but debounce expensive API calls.
- Use the provided `abortSignal` in image uploads to handle cancellations.
