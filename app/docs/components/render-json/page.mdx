# RenderJSON

The `RenderJSON` component renders Tiptap JSON content generated from `editor.getJSON()` or `getEditorContent().json`.

It is the recommended renderer for saved editor content because it preserves the original Tiptap document structure, including editor-specific nodes such as tables, images, task lists, and code blocks.

## Import

```tsx
import { RenderJSON } from "tiptap-react-ui";
```

## Signature

```tsx
function RenderJSON(props: RenderJSONProps): JSX.Element | null;
```

## Usage

```tsx
import { RenderJSON } from "tiptap-react-ui";

export default function Post({ content }) {
  return <RenderJSON content={content} />;
}
```

## Using Editor Content

A common workflow is saving content with `editor.getJSON()` or `getEditorContent().json` and rendering it later.

```tsx
const json = editor.getJSON();

<RenderJSON content={json} />;
```

or

```tsx
const { getEditorContent } = useTiptapEditor();

<RenderJSON content={getEditorContent().json} />;
```

## Props

| Prop                      | Type                            | Default     | Description                                                                                                                             |
| ------------------------- | ------------------------------- | ----------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| `content`                 | `JSONContent`                   | Required    | Tiptap JSON document from `editor.getJSON()`.                                                                                           |
| `immediatelyRender`       | `boolean`                       | `false`     | Controls immediate rendering. Defaults to `false` for SSR-sensitive pages.                                                              |
| `className`               | `string`                        | -           | Classes applied to the root `.tr-editor` rendered output wrapper.                                                                       |
| `mode`                    | `"light" \| "dark" \| "system"` | `"system"`  | Applies light or dark mode styles to the rendered output.                                                                               |
| `enableTableOfContents`   | `boolean`                       | `false`     | Shows a generated table of contents beside rendered JSON content.                                                                       |
| `tableOfContentsPosition` | `"left" \| "right"`             | `"right"`   | Controls which side the rendered JSON TOC appears on.                                                                                   |
| `fallback`                | `React.ReactNode`               | `undefined` | Fallback content displayed until the editor is initialized. This can be any valid React node, such as text, JSX, or a loading skeleton. |

## Styling

`className` applies to the root `.tr-editor` wrapper. Use it for layout, typography, spacing, background, and theme context.

```tsx
<RenderJSON content={content} className="prose dark:prose-invert max-w-none" />
```

`RenderJSON` no longer exposes public `contentClassName`, `editorsClassName`, or `_height` props. The internal read-only editor uses fixed safe defaults:

```txt
max-h-none overflow-y-visible bg-transparent
```

This lets the rendered content grow naturally, keep overflow visible, and adopt the visual styling from the root wrapper.

## Recommended Storage Shape

Store JSON as the source of truth when users will edit the content again or when you want the most faithful read-only rendering. Store HTML alongside it only when you need display-ready markup for search indexing, email, CMS integrations, or external systems.

```ts
type StoredDocument = {
  id: string;
  content: JSONContent;
};
```

## Examples

### Basic Rendering

```tsx
<RenderJSON content={content} />
```

### Typography Styling

```tsx
<RenderJSON content={content} className="prose dark:prose-invert max-w-none" />
```

### Dark Mode Rendering

```tsx
<RenderJSON content={content} mode="dark" />
```

Use `mode="system"` when the renderer should follow your app's global dark-mode scope.

### Table Of Contents

```tsx
<RenderJSON
  content={content}
  enableTableOfContents
  tableOfContentsPosition="right"
/>
```

The TOC is generated from the rendered Tiptap JSON state. It is not available in `RenderHTML` because raw serialized HTML does not include the live Tiptap TOC state.

### Next.js / SSR

```tsx
<RenderJSON content={content} />
```

### Custom Layout

```tsx
<RenderJSON
  content={content}
  className="mx-auto max-w-4xl rounded-lg bg-background p-6"
/>
```

## Common Use Cases

### Blog Posts

```tsx
<RenderJSON content={post.content} />
```

### CMS Content

```tsx
<RenderJSON content={page.content} />
```

### Documentation Pages

```tsx
<RenderJSON content={docs.content} />
```
