# RenderHTML

The `RenderHTML` component is a utility for rendering HTML content generated from `editor.getHTML()`.

It is ideal for displaying saved editor content in blogs, CMS pages, documentation sites, article pages, and other read-only content experiences.

## Import

```tsx
import { RenderHTML } from "tiptap-react-ui";
```

## Signature

```tsx
function RenderHTML(props: RenderHTMLProps): JSX.Element;
```

## Usage

```tsx
<RenderHTML content={post.html} className="prose max-w-none" />
```

## Using Editor Content

A common workflow is saving content with `editor.getHTML()` or get html content from `getEditorContent` and rendering it later.

```tsx
const html = editor.getHTML();

<RenderHTML content={html} />;
```

or

```tsx
const {editor,getEditorContent} = useTiptapEditor()

<RenderHTML content={getEditorContent().html} />;
```

## Props

| Prop        | Type                       | Default  | Description                                  |
| ----------- | -------------------------- | -------- | -------------------------------------------- |
| `content`   | `HTMLContent`              | Required | HTML string generated by `editor.getHTML()`. |
| `className` | `string`                   | -        | Classes applied to the wrapper element.      |
| `sanitize`  | `(html: string) => string` | -        | Optional sanitizer called before rendering.  |

## Security

When rendering user-generated HTML, it is recommended to sanitize content before displaying it.

Example:

```tsx
import DOMPurify from "dompurify";

<RenderHTML content={content} sanitize={DOMPurify.sanitize} />;
```

> **Note**
>
> HTML generated directly from your own editor content is generally safe if you fully control the editor configuration. If content can be modified by users or external systems, sanitization is strongly recommended.

## Examples

### Basic Rendering

```tsx
<RenderHTML content={content} />
```

### Tailwind Typography

```tsx
<RenderHTML content={content} className="prose dark:prose-invert max-w-none" />
```

### Blog Post

```tsx
<RenderHTML
  content={post.content}
  className="prose lg:prose-lg dark:prose-invert mx-auto"
/>
```

### CMS Content

```tsx
<RenderHTML content={page.content} className="prose max-w-none" />
```

### With Sanitization

```tsx
import DOMPurify from "dompurify";

<RenderHTML content={content} sanitize={(html) => DOMPurify.sanitize(html)} />;
```

## When to Use RenderHTML

Use `RenderHTML` when:

- Your content is stored using `editor.getHTML()`
- You need simple HTML rendering
- Content is consumed by external systems
- You're building blogs or CMS platforms
- HTML output is your primary storage format

## RenderHTML vs RenderJSON

| Feature                     | RenderHTML         | RenderJSON         |
| --------------------------- | ------------------ | ------------------ |
| Input                       | `editor.getHTML()` | `editor.getJSON()` |
| Content Format              | HTML               | Tiptap JSON        |
| Supports Sanitization       | ✅                 | ❌                 |
| Easier External Integration | ✅                 | ❌                 |
| Preserves Editor Structure  | ❌                 | ✅                 |
